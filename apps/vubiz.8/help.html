How is V8 different?

  • It is mobile/touch friendly. This feature will increasingl become important as our HTML content collection grows.
  • It offers enhanced security by using user controlled passwords, enhanced access options and optionally configured to run using SSL.
  • It is only for learning. Facs/Mgrs/Adms will link to V5 (V8a eventually) for Reports, etc which will NOT be mobile/touch friendly.
  • Sign In pages (ChAccess Pages) are embedded in V8.
  • Allows Advanced Learner Uploads and the ability to tie learners to a particular Facilitator for reporting.
  • A V8 account can be configured to allow a learner to Add one or more Guest Leaners. And example is CFIB's members who are tyipcally small business owners. Each CFIB member/learner can add one or more of their employees into the system to access all or some of the account catalogue.
  • V8 is part of our growing family of Vubiz "apps". Each app uses a "Profile" table that addresses their unique services. This table allows app specific info to enhance the service, ie like the embedded "Sign In pages"

Which accounts can and cannot use V8?

  • V8 is targetted to our next generation "clean" channels, ie those that do not use custom functionality.
  • V8 will support content and child accounts created via Ecommerce however ecommerce is not part of V8.
  • While G2 Parent/Child Sites are supported, the legacy G1 sites are NOT supported.
  • Corporate sites are NOT supported
  • V8 only supports content using our SCORM RTE. This includes FX, XX and Z but does NOT support the leagacy F nor X content.
  • Existing channels that match this criteria can be ported to V8.

When will V8 be available.

  • CFIB is our first V8 custmer. They are a good account to launch this service as they have a high profile, and only use the core features.
  • V8 with CFIB should be live by September.
  • V8 with ecommerce using the VUBIZ house account should be functional by the end of the year.
  • V8 should be availble for new customers or the migration of existing customer by early 2015.

Configuring a V8 Account

This is done on the Customer Table using V5. Since V8 is a Channel Service there are a set of flags that need to be addressed.
  • Is V8 : Does this account meet the criteria to be V8, ie is it a clean channel using RTE content.
  • Is G2 Parent : If this is a Group Account is it a G2 Parent or Child
  • Use V8 Uploads with Reports To : Does this V8 client want to use the Advanced Upload to manage their learners?
  • Use V8 Guest System : can a learner invite a guest to access the account. Typically use in Associations like CFIB.
  • To use Single Sign On (SSO) enable Auto Enroll. This implies that Members cannot access the site directly, they must come in with a Customer Guid.

Access Options

There are three ways to enter the V8 service (configurable):
  1. SSO - Single Sign On

    This is available for clients who authenticate their users behind their own firewall. When they are redirected to us they do NOT need a password but they will need an Account Guid. These guids are unique to each Customer Id and would be given out to the client by Helen.  CFIB for example would have an SSO link to our site as follows:
    http://cfib.vubiz.com?custGuid=abcdef12-1234-5554-abab-3333&learnerId=123456
    ...where 123456 is the CFIB Member No and the custGuid is issued by Vubiz
    For Security reasons we all want to keep the GUIDS as confidential as possible.
    Ideally we want the SSO to be a form post to V8 but that will depend on ths client's setup.

  2. Direct Access

    We will optionally allow clients direct access by passing in all these variables, typically via the URL.
    Soon we will enhance security by offering a V8 direct access option: none | url | form.
    V5 allows URL direct access and V8 will start by allowing it but soon we can either disable this or only support direct access via a form post.
    A typical form post to CFIB would look like this:
    http://cfib.vubiz.com?leanerId&learnerId=123456&password=password;
  3. Sign In Form

    When the service does not recognize either SSO or Direct Access it will render a sign in page.
    Using profile data the Sign In page will be able to replicate the older ChAcess service.
    If V8 does not know the profile it will render a Customer Id text box.

Future Considerations

  • Technically no-one other than the learner should ever know their password. Learners should recreate their own rather than look to VUBIZ to tell them the password. We have an app for that purpose.
  • For V8 clients using existing reports, we will change V5 channel term Password to Learner ID as we do with Corporate - BUT unlike Corporate passwords, we would never make V8 passwords available anywhere.
